Use this workflow when the task needs a short product-intent brief before the normal execution brief.

Task context: $ARGUMENTS

Steps:
1. Inspect the repository and any task context that already exists.
2. Frame the problem being solved, the user or operator affected, and the desired outcome.
3. Capture constraints, success criteria, explicit non-goals, and open assumptions.
4. Keep the output lightweight so it can feed the normal execution-brief workflow rather than replace it.
5. Escalate if missing product intent would materially change scope or architecture.

Output format:
- problem
- affected user or operator
- desired outcome
- constraints
- success criteria
- explicit non-goals
- open assumptions
